오늘 배운 기준으로 내 업무 문구를 점검합니다. 아래 체크리스트를 확인 후 한 가지부터 실행.
실습  ①  ·  문제

이 버튼, 어떻게 고칠까요?

⏱ 2분
상황: 전시 예약 완료 직전 확인 팝업
"《미래의 미술》 전시를 예약하시겠습니까?"

확인 취소
힌트: P1 버튼 원칙 — "동사+목적어" / P2 KRDS — "맥락을 포함한 버튼 쌍"
생각해볼 것: "확인"을 누르면 무슨 일이? "취소"를 누르면?
✏️
혼자 먼저 생각해보고, 옆 사람과 의견을 나눠보세요.
"확인" 대신 어떤 단어가 들어가면 더 명확할까요?
실습  ①  ·  해설

이 버튼, 어떻게 고칠까요? — 해설

원래 버튼
확인 취소
❌ "확인"이 예약인지 단순 확인인지 모름
❌ "취소"가 예약 취소인지 팝업 닫기인지 모름
개선된 버튼
예약 완료하기 돌아가기
✓ 클릭하면 예약이 완료됨을 앎
✓ "돌아가기"는 예약 취소가 아닌 이전 화면
💡
버튼 원칙: [주요 동작] + 하기 vs [이전 상태로] + 돌아가기
"취소"는 한국어에서 예약 취소로 오해받기 쉽습니다. "돌아가기"가 더 안전합니다.
③ 심리적안전망 원칙 적용
실습  ②  ·  문제

이 에러 메시지, 어떻게 고칠까요?

⏱ 2분
상황: 예약 신청 중 결제 오류 발생
"결제 처리 중 오류가 발생하였습니다.
Error Code: PAY_TIMEOUT_001
고객센터에 문의하시기 바랍니다."
힌트: P1 나쁜 글 3패턴 — 기술 중심 메시지 / P2 KRDS 에러 메시지 3단 구조
생각해볼 것: 사용자가 지금 느끼는 감정은? 다음에 무엇을 해야 하는지 알 수 있는가?
✏️
KRDS 에러 메시지 3단 구조를 기억하세요:
① 무슨 일인지 + ② 사용자 잘못이 아님 + ③ 사용자가 취할 다음 행동
실습  ②  ·  해설

에러 메시지 — 해설

원래 메시지
"결제 처리 중 오류가 발생하였습니다.
Error Code: PAY_TIMEOUT_001
고객센터에 문의하시기 바랍니다."
❌ 에러 코드 — 사용자에게 무의미
❌ "고객센터"가 어디인지 모름
❌ 내 잘못인지 시스템 문제인지 모름
KRDS 기준 개선
"결제 중 잠깐 문제가 생겼어요.
카드에는 청구되지 않았으니 안심하세요.
잠시 후 다시 시도하거나, 전화(02-2124-8800)로 문의해주세요."
✓ ① 무슨 일인지 (결제 중 문제)
✓ ② 잘못 아님 (청구 없음 안심)
✓ ③ 다음 행동 (재시도 or 전화)
💳
결제 오류에서 "카드에 청구되지 않았습니다"는 가장 중요한 한 줄입니다.
사용자의 가장 큰 불안(돈 빠져나감?)을 먼저 해소하세요.
③ 심리적안전망 원칙 적용
실습  ③  ·  문제

이 이메일 제목, 어떻게 고칠까요?

⏱ 2분
상황: 관람객에게 보내는 예약 리마인더 이메일
제목: "[서울시립미술관] 관람 안내드립니다"

본문: 내일 예약하신 전시 관람 일정을 안내드립니다. 내일 오후 2시에...
힌트: P3 발견 ⑤ 공지 제목 패턴 / 이메일 제목만 봤을 때 열어볼지 판단 가능한가?
생각해볼 것: 받는 사람이 모르는 사람이라면? 제목만 봐서 무슨 내용인지 알 수 있나?
✏️
이메일 제목 공식: [SeMA] + [무엇이] + [언제] + [어떻게]
"안내드립니다"는 제목이 아닙니다. 무엇을 안내하는지가 제목입니다.
실습  ③  ·  해설

이메일 제목 — 해설

원래 제목
[서울시립미술관] 관람 안내드립니다
❌ 무슨 관람인지 모름
❌ 언제인지 모름
❌ 열어봐야 할 이유가 없음
개선된 제목 (여러 버전)
[SeMA] 내일 오후 2시, 예약하신 전시입니다
[SeMA] 《미래의 미술》 — 내일 오후 2시 관람
[SeMA] 내일 예약 리마인더 — 서소문본관 오후 2시
✓ 전시명, 날짜, 시간 포함
✓ 제목만 봐도 열어볼지 판단 가능
📧
세 버전 모두 좋습니다. 정답은 없고, 명확함이 있을 뿐입니다.
여러분이 작성한 버전도 이 기준을 충족한다면 맞습니다.
② 정보핵심화 원칙 적용
실습  ④  ·  문제

이 공문 안내문, 어떻게 고칠까요?

⏱ 2분
상황: 4월 교육 프로그램 참가 신청 공지 (직원이 작성한 공문 스타일)
"2026년 4월 교육 프로그램 참가 신청에 관하여 아래와 같이 공지하오니,
관련 양식을 구비하시어 신청 기간 내에 접수하여 주시기 바랍니다."
힌트: P1 나쁜 글 패턴 — 추상적 표현, 역할 중심 문체 / P2 규칙 — "기한 내" 금지
생각해볼 것: "신청 기간 내"가 언제인지 알 수 있는가? "양식을 구비"는 무슨 의미인가?
✏️
공문체 도입부 + 모호한 기간 + 역할 중심 문체 — 세 가지 문제가 한 문장에 있습니다.
읽은 사람이 바로 행동할 수 있는 문장으로 바꿔보세요.
실습  ④  ·  해설

공문 안내문 — 해설

원래 공문 스타일
"2026년 4월 교육 프로그램 참가 신청에 관하여 아래와 같이 공지하오니, 관련 양식을 구비하시어 신청 기간 내에 접수하여 주시기 바랍니다."
❌ 도입부 — "아래와 같이 공지하오니" 불필요
❌ "신청 기간 내" — 언제인지 모름
❌ "양식을 구비" — 무슨 양식, 어디서?
❌ "접수하여 주시기 바랍니다" — 어디에?
KRDS 기준 개선
"4월 교육 프로그램 신청을 시작합니다.
4월 7일(월)부터 4월 18일(금) 오후 6시까지
SeMA 홈페이지(sema.seoul.go.kr)에서 신청서를 작성해 제출해주세요."
✓ 핵심 내용 먼저 (신청 시작 공지)
✓ 구체적 기간 (KRDS 날짜 형식)
✓ 방법·수신처 명확
✓ "아래와 같이" 도입부 제거
📋
공문 안내문 공식: [무엇을 신청] + [언제까지] + [어디서/어떻게]
"아래와 같이 공지하오니"는 지워도 됩니다. 그게 시작입니다.

내일부터 바로  ·  개인 체크리스트

강연 후 바로 쓰는 UX라이팅 체크리스트

□
수신자 행동 확인 — 이 글을 읽고 상대방이 바로 행동할 수 있는가?
행동이 불분명하면 날짜·장소·방법 중 빠진 것을 추가하세요
□
날짜·시간 형식 — "4/15 18:00" 대신 "4월 15일(화) 오후 6시"로 썼는가?
숫자 단독 표기는 오해 유발 — 한글로 풀어 쓰기
□
버튼·링크 텍스트 — 동사로 시작하고, 클릭 후 무슨 일이 일어나는지 알 수 있는가?
"확인" → "예약 완료하기"  /  "더보기" → "전시 전체 보기"
□
에러·안내 메시지 — [무슨 일] + [내 잘못 아님] + [다음 행동]이 모두 있는가?
연락처까지 포함하면 문의 전화가 줄어듭니다
□
도입부 제거 — "안내드립니다", "아래와 같이 공지하오니" 같은 빈 문장을 삭제했는가?
첫 문장에서 바로 핵심 내용으로 시작하세요
□
플레이스홀더 vs 레이블 — 입력창에 레이블이 따로 있는가? (플레이스홀더만 쓰지 않았는가?)
입력 시작 시 힌트가 사라지면 사용자가 맥락을 잃습니다
□
알림·팝업 텍스트 — 모달 헤더에 목적이 명시되었는가? 토스트는 1줄인가?
"알림" 헤더 + "확인" 버튼 조합은 금지 — "예약을 취소하시겠어요?" + "예약 취소 / 돌아가기"
□
경어체 일관성 — 한 문서·화면 안에서 합쇼체와 해요체가 섞이지 않았는가?
"완료되었습니다"(합쇼체) + "확인해요"(해요체) 혼용 금지 — 하나로 통일
📋
