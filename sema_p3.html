발견  ③

에러·오류 안내 부재 또는 기술 언어

발견된 문제
  • 예약 실패 시 "오류가 발생했습니다" — 이유와 다음 행동 없음
  • 세션 만료 시 안내 없이 로그인 페이지로 이동
  • 첨부 파일 용량 초과 시 기술 코드만 표시
KRDS 기준 개선안
  • "예약 중 문제가 생겼어요. 잠시 후 다시 시도하거나 전화로 문의해주세요."
  • "이용 중 자리를 비운 사이 연결이 끊겼어요. 다시 로그인해주세요."
  • "파일은 10MB 이하만 첨부할 수 있어요. 파일 크기를 줄여 다시 올려주세요."
🛟
에러 메시지 공식: ① 무슨 일인지 + ② 사용자 잘못이 아님을 암시 + ③ 다음 행동
"잠시 후" "다시 시도" "문의해주세요" — 이 세 가지만 있으면 됩니다.
발견  ④

날짜·시간 표기 비일관성

위치 발견된 표기 문제점 KRDS 기준
전시 기간 2026.3.1~5.31 물결표, 요일 없음 3월 1일(일)~5월 31일(일)
예약 마감 4/14 18:00까지 24시간제, 요일 없음 4월 14일(화) 오후 6시까지
관람 시간 10:00~20:00 24시간제, 외국식 오전 10시~오후 8시
공지사항 2026-03-15 ISO 8601 형식, 어색 2026년 3월 15일
🗓️
즉시 가능한 개선: "18:00", "10:00~20:00", "2026.3.1" 등을 검색해 KRDS 형식으로 일괄 교체
한 번 정해두면 이후에는 자동으로 통일됩니다.
발견  ⑤

공지사항 제목 패턴 — 행동 유도 부족

현재 공지사항 제목 패턴
  • SeMA 전시 안내 공지
  • 이용 안내 사항 알림
  • 홈페이지 점검 안내
  • 기관 관련 공지사항
❌ 무엇이 바뀌었는지 모름
❌ 읽어야 할 이유가 없음
개선된 제목 제안
  • 《전시명》 — 4월 15일 오전 10시부터 관람 가능
  • 예약 시스템 변경 — 5월부터 앱으로만 예약
  • 4월 1일(수) 오전 2~6시 홈페이지 점검
  • 3월 프로그램 접수 — 4월 1일까지
✓ 제목만 봐도 내용을 알 수 있음
✓ 날짜와 변경 내용이 명확
📢
공지사항 제목 공식: [무엇이] + [언제] + [어떻게 바뀌는지]
목록에서 제목만 스캔해도 읽을지 말지 판단할 수 있어야 합니다.

예약 완료 화면 — 버튼과 안내 문구

현재 화면 (재현)
"전시 예약이 완료되었습니다."
확인 취소
❌ 예약 번호, 일시, 장소 없음
❌ "확인"이 무엇인지 모름
❌ "취소"가 예약 취소인지 닫기인지
개선안
"예약이 완료됐어요! 🎉
4월 20일(일) 오후 2시, 서울시립미술관 서소문본관
예약번호 2026-04-1234를 저장해두세요."
예약 확인서 저장 전시 더 보기
✓ 예약 정보 모두 포함
✓ 버튼 결과 예측 가능
✓ 사람처럼 축하해줌
🎊
완료 화면은 사용자가 가장 기쁜 순간입니다. 그 기쁨에 함께하세요.
기술적 확인보다, 사람처럼 축하하는 말이 기억에 남습니다.

예약 실패 에러 메시지

현재 에러 메시지
"오류가 발생했습니다.
관리자에게 문의하세요."
❌ 무슨 오류인지 모름
❌ "관리자"가 누구인지 모름
❌ 어떻게 문의하는지 모름
❌ 내 잘못인지 모름
KRDS 기준 개선안
"예약 중 문제가 생겼어요.
잠시 후 다시 시도하거나,
전화(02-2124-8800)로 문의해주세요."
✓ 상황을 이해할 수 있음
✓ 사용자 잘못이 아님을 암시
✓ 다음 행동 2가지 제시
✓ 연락처 포함
📞
에러 메시지에서 전화번호는 강력한 안전망입니다.
디지털이 실패했을 때 사람이 도울 수 있다는 것을 알려주면 신뢰가 유지됩니다.

이메일 제목 — 공지와 예약 안내

현재 이메일 제목 패턴
  • [SeMA] 예약 안내
  • [서울시립미술관] 공지사항
  • [안내] 이용 안내드립니다
❌ 제목만 봐서는 내용 불명
❌ 열지 않아도 될 것 같은 느낌
개선된 이메일 제목
  • [SeMA] 4월 20일(일) 예약이 확정됐어요
  • [SeMA] 5월 1일부터 예약 방식이 바뀝니다
  • [SeMA] 오늘 예약하신 전시, 내일입니다
✓ 제목만 봐도 읽을지 판단 가능
✓ 날짜와 핵심 내용 포함
📨
이메일 제목은 열어볼지 말지를 결정하는 첫 문장입니다.
"안내드립니다"는 제목이 아닙니다. 무엇을 안내하는지를 써야 합니다.
개선 로드맵  ·  Phase 1

즉시 실행 가능한 3가지 (이번 주)

즉시
버튼 텍스트 교체 — "확인" → "예약 완료하기", "취소" → "예약 취소하기"
예상 효과: 클릭률 향상, 실수로 인한 취소 감소
즉시
날짜 표기 통일 — 사이트 전체에서 "18:00" → "오후 6시", "4/15" → "4월 15일(화)"
예상 효과: 관람객 혼란 감소, 일정 앱 복사 용이
즉시
에러 메시지 개선 — "오류가 발생했습니다" → 상황 설명 + 전화번호 포함
예상 효과: 고객 문의 전화 감소, 사용자 이탈 방지
⚡
세 가지 모두 HTML 구조 변경 없이 텍스트만 수정하면 됩니다.
개발자 없이 담당자가 직접 할 수 있습니다.
개선 로드맵  ·  Phase 2

중기 개선 과제 (1~3개월)

1개월
공지사항 제목 가이드 수립
"[무엇이] + [언제] + [어떻게]" 형식으로 작성 가이드 문서화. 새 공지 작성 시 체크리스트 활용.
✓ 담당자 매뉴얼로 배포 가능
1개월
이메일 템플릿 재작성
예약 확인·취소·리마인더 이메일 제목과 본문을 KRDS 기준으로 재작성. 날짜·장소·예약번호 필수 포함.
✓ 재사용 가능한 템플릿
3개월
빈 화면·로딩 메시지 정비
예약 내역 없음, 검색 결과 없음, 로딩 중 등 빈 상태 문구를 KRDS 공식으로 통일.
✓ 전체 사이트 일관성 향상
3개월
텍스트 선택 차단 코드 제거
접근성 위반 및 실효성 없는 우클릭·선택 차단 코드를 제거. 개발자 작업 필요.
✓ 접근성 개선 + 관람객 편의
🗺️
Phase 1(즉시)을 완료하면 가장 눈에 띄는 개선 효과를 바로 확인할 수 있습니다.
Phase 2는 그 기반 위에서 체계적으로 진행합니다.
P3  ·  종합 정리

5가지 발견 요약

발견 현재 문제 영향 조치
① 선택 차단 텍스트 복사·우클릭 불가 접근성 위반 코드 제거 (개발자)
② 버튼 모호성 "확인" "취소" 맥락 없음 실수 증가 텍스트 교체 (즉시)
③ 에러 메시지 기술 언어 + 다음 행동 없음 이탈, 전화 급증 문구 재작성 (즉시)
④ 날짜 혼재 3가지 이상 형식 혼용 신뢰도 하락 전체 통일 (즉시)
⑤ 공지 제목 내용 없는 제목 공지 열람률 저하 가이드 수립 (1개월)
P3  ·  기대 효과

작은 변화, 큰 차이

📉
문의 전화 감소
에러 메시지와 날짜 표기가 명확해지면 "언제예요?" "어떻게 해요?" 전화가 줄어듭니다.
✅
예약 완료율 향상
버튼 텍스트가 명확해지면 실수로 취소하거나 중간에 이탈하는 비율이 낮아집니다.
🏆
기관 신뢰도 향상
일관된 언어와 명확한 안내는 "전문적인 기관"이라는 인상을 줍니다.
💡
UX라이팅 개선은 디자인 리뉴얼 없이도 가능합니다.
텍스트만 바꿔도 사용자 경험은 달라집니다.
