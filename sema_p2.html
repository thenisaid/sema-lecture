01  ·  기준의 필요성

일관성 없는 텍스트가 신뢰를 깎는다

기준 없을 때 생기는 문제
  • 같은 기능을 페이지마다 다른 이름으로 — "예약" vs "신청" vs "접수"
  • 날짜 표기 혼재 — "2026.4.1" "4월 1일" "04/01"
  • 버튼이 어떤 곳은 "확인", 어떤 곳은 "완료", 어떤 곳은 "OK"
  • 에러 메시지가 기술 용어 그대로 노출
기준이 있을 때의 효과
  • 사용자가 처음 보는 화면도 익숙하게 느낌
  • 여러 담당자가 작성해도 같은 목소리
  • 새 직원 온보딩 시간 단축
  • 기관 브랜드 신뢰도 향상
⚖️
기준은 창의성을 막지 않습니다. 기준은 매번 처음부터 생각하지 않아도 되는 자유를 줍니다.
날짜를 어떻게 쓸지 고민하지 않아도 되면, 더 중요한 내용에 집중할 수 있습니다.
02  ·  KRDS란 무엇인가

한국 정부 디자인 시스템 (Korea Republic Design System)

🏛️
만든 곳
행정안전부 주도, 공공기관 디지털 서비스 일관성을 위해 제정
📐
무엇을 정의하나
색상·타이포그래피·컴포넌트·아이콘과 함께 UX라이팅 규칙 포함
🎯
누가 써야 하나
정부·공공기관·지자체가 운영하는 디지털 서비스 — SeMA 포함
📖
KRDS UX라이팅 가이드라인은 날짜·시간 표기, 버튼 텍스트, 에러 메시지, 빈 화면 문구에 대한 구체적인 기준을 제공합니다.
오늘은 SeMA에 바로 적용할 수 있는 핵심 4가지를 살펴봅니다.
03  ·  규칙 1

날짜·시간 표기 기준

날짜
연·월·일 순서, 괄호로 요일 표기
✗ 4/15 (화)  →  ✓ 2026년 4월 15일(화)
시간
오전/오후 + 시:분 형식, 24시간제 사용 안 함
✗ 18:00  →  ✓ 오후 6시  또는  ✓ 오후 6시 30분
기간
~(물결표) 대신 "부터 ~ 까지" 명시
✗ 4.1~4.30  →  ✓ 4월 1일부터 4월 30일까지
마감
"기한 내" 금지, 구체적 날짜+시간 명시
✗ 기한 내 제출  →  ✓ 4월 15일(화) 오후 6시까지 제출
03  ·  규칙 2

버튼 텍스트 기준

형식
동사+목적어, 명사형 종결 허용 (명령조 X)
✗ 삭제하라 / 클릭!  →  ✓ 예약 취소 / 파일 다운로드
길이
최대 6자 이내 권장 (모바일 기준)
✗ 지금 바로 예약 신청하기  →  ✓ 예약 신청
쌍
확인/취소 쌍은 반드시 맥락을 포함
✗ 확인 / 취소  →  ✓ 예약 완료 / 돌아가기
🔘
KRDS 기준: 버튼은 클릭 결과를 예측할 수 있어야 합니다.
"확인"은 무엇을 확인하는지 모릅니다. "예약 완료"는 클릭하면 예약이 완료된다는 것을 압니다.
03  ·  규칙 3

에러 메시지 기준

기술 중심 (피해야 할 것)
"오류 코드 404: 요청한 리소스를 찾을 수 없습니다."
❌ 사용자가 이해 못함
❌ 다음 행동을 알 수 없음
❌ 불안감만 줌
사람 중심 (KRDS 기준)
"찾으시는 페이지가 없어요.
주소를 다시 확인하거나 처음으로 돌아가세요."
✓ 상황을 이해할 수 있음
✓ 다음 행동 2가지 제시
✓ 사람처럼 말함
🛟
KRDS 에러 메시지 3요소: ① 무슨 일이 생겼는지 + ② 사용자 잘못이 아님 + ③ 다음에 무엇을 하면 되는지
"죄송합니다. 일시적인 오류입니다. 잠시 후 다시 시도해주세요." → 3요소 충족 예시
03  ·  규칙 4

빈 화면(Empty State) 기준

피해야 할 빈 화면
✗ 나쁜 예
"데이터가 없습니다."
기술 언어. 사용자가 무엇을 해야 할지 모름.
권장하는 빈 화면 (KRDS 기준)
✓ 좋은 예
"예약 내역이 아직 없어요.
전시를 둘러보고 첫 예약을 해보세요."
상황 설명 + 행동 안내
📭
빈 화면은 기회입니다. 사용자가 막막해하지 않도록 다음 행동을 안내하세요.
KRDS 공식: "[상태 설명]. [다음 행동 안내]."
03  ·  실제 적용

SeMA에서 자주 쓰는 날짜·시간 표기

상황 ❌ 피해야 할 표기 ✓ KRDS 기준
전시 기간 2026.3.1~5.31 2026년 3월 1일(일)부터 5월 31일(일)까지
예약 마감 4/14 18:00까지 4월 14일(화) 오후 6시까지
관람 시간 10:00~20:00 (월 휴관) 오전 10시~오후 8시 (매주 월요일 휴관)
접수 기간 3월 중 접수 가능 3월 1일(토)부터 3월 31일(월) 오후 6시까지 접수
📅
날짜 표기 하나를 통일하면 전체 사이트의 신뢰도가 올라갑니다.
가장 먼저 할 일: "기한 내"라는 단어를 검색해서 모두 구체적 날짜로 교체
03  ·  실제 적용

링크와 버튼 텍스트 교체하기

위치 ❌ 현재 ✓ KRDS 기준
예약 버튼 확인 예약 완료하기
취소 버튼 취소 예약 취소하기
더보기 링크 여기를 클릭하세요 전시 목록 전체 보기
안내 링크 자세히 보기 관람 안내 자세히 보기
🔗
링크 텍스트 규칙: "여기를 클릭하세요" 금지. 링크 텍스트만 읽어도 목적지를 알아야 합니다.
스크린리더 사용자는 링크만 추출해서 읽습니다 — "여기"만 나열되면 의미를 알 수 없습니다.

P2  ·  마무리

오늘 배운 것 정리

핵심 요약
  • 기준이 있으면 담당자가 바뀌어도 같은 목소리
  • KRDS는 공공기관을 위한 공식 가이드라인
  • 날짜: 연·월·일 + 요일 + 오전/오후
  • 버튼: 동사+목적어, "확인" 사용 금지
  • 에러: 상황 + 잘못 아님 + 다음 행동
즉시 할 수 있는 것
  • 사이트에서 "기한 내" 검색 → 날짜로 교체
  • "여기를 클릭" 검색 → 목적지 텍스트로 교체
  • 버튼 텍스트 목록 정리 → 쌍으로 재검토
  • 에러 메시지 3요소 체크리스트 적용
