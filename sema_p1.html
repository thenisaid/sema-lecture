01  ·  UX라이팅이란

제품 안에 살아있는 텍스트를 설계하는 일

UX라이팅이 적용되는 곳
  • 버튼 레이블 — "예약하기" "취소" "다운로드"
  • 에러 메시지 — 오류 안내, 실패 시 다음 행동
  • 안내 문구 — 빈 화면, 로딩, 완료 화면
  • 이메일 제목과 본문
  • 입력 필드 플레이스홀더
UX라이팅의 목표
  • 사용자가 다음 행동을 바로 알 수 있게
  • 시스템이 아닌 사람의 언어로
  • 실수를 예방하거나 회복을 돕게
  • 브랜드의 목소리와 톤을 일관되게
💬
UX라이팅은 "어떤 말을 쓸까"가 아니라 "이 상황에서 사용자에게 무엇이 필요한가"에서 시작합니다.
디자이너가 만든 화면 안에서, 사용자의 다음 행동을 안내하는 텍스트 설계입니다.
01  ·  개념 비교

UX라이팅 vs 마케팅 카피라이팅

구분 UX라이팅 마케팅 카피라이팅
목적 막히지 않고 다음 행동으로 이동 관심을 끌고 구매·참여를 유도
위치 제품 내부 (버튼, 에러, 안내) 광고, 랜딩페이지, 배너
스타일 짧고 명확, 기능 중심 설득적, 감성적, 눈에 띄는 언어
성공 기준 사용자가 글을 의식하지 못하고 진행 사용자가 글을 읽고 반응
🎯
최고의 UX라이팅은 눈에 보이지 않는 글입니다.
사용자가 읽었는지조차 모르고 자연스럽게 다음 단계로 넘어갔다면 성공입니다.
02  ·  우리 주변의 UX라이팅

이미 경험하고 있는 UX라이팅

📱
카카오페이
"5초 안에 이체 완료"
과정이 아닌 결과를 먼저
✓ 사용자 관점
🚀
토스
"만약 카드를 잃어버렸다면, 지금 바로 잠글 수 있어요."
공감에서 시작
✓ 인간적인 어조
🏛️
정부24
"처리 완료됩니다"
수동태 + 모호한 미래
✗ 맥락 부족
흔한 시스템 언어
"처리 중입니다. 잠시 기다려 주십시오."
❌ 얼마나 기다려야 하는지 알 수 없음
사람의 언어
"예약을 확인하고 있어요. 10초 정도 걸려요."
✓ 예상 시간 + 친근한 어조
02  ·  미술관 현장

우리는 이미 UX라이팅을 하고 있다

📧
예약 확인 이메일
관람객에게 보내는 예약 안내 메일의 제목, 본문, 안내 사항
⚠️
오류·안내 팝업
예약 실패, 결제 오류, 일정 변경 시 사용자에게 보이는 문구
🔘
버튼 텍스트
웹사이트의 "예약" "신청" "확인" "취소" — 모두 UX라이팅
💡
미술관 직원이 매일 작성하는 공문, 이메일, 안내 팝업은 모두 UX라이팅입니다.
UX라이팅을 배운다는 것은 기존 업무를 사용자 관점으로 다시 보는 것입니다.
03  ·  나쁜 글의 패턴

읽어도 모르는 글이 되는 3가지 이유

🌫️
추상적 표현
"기한 내 접수 요망"
→ 기한이 언제? 어디로?
읽고 전화해야 함
🏢
역할 중심 문체
"이용 불가 처리될 수 있음을 알려드립니다"
→ 시스템이 말하는 것처럼
사람 같지 않음
💻
기술 중심 메시지
"오류 코드 500: 서버 오류가 발생했습니다"
→ 사용자가 할 수 있는 것이 없음
불안감만 줌
🔍
나쁜 글의 공통점: 작성자가 쓰기 편한 것을 우선했습니다.
"기한 내"는 쓰기 편하지만, "4월 15일 오후 6시까지"가 사용자에게 유용합니다.
→ 이 세 패턴이 KRDS 3대 원칙(무번역·정보핵심화·심리적안전망)이 해결하려는 바로 그 문제입니다. (S11)
03  ·  좋은 글의 조건

읽으면 바로 행동이 보이는 글

📍
구체적이다
날짜, 장소, 방법, 수신처가 명확
"4월 15일 오후 6시까지 이메일로"
✓ 5W1H 충족
👆
행동을 안내한다
읽은 후 다음 단계가 명확
"보내주세요" "클릭하세요" "확인해주세요"
✓ 다음 행동 명시
🤝
사람처럼 말한다
시스템이 아닌 사람의 말투
"죄송합니다, 잠시 후 다시 시도해주세요"
✓ 신뢰감 형성
✏️
글을 쓴 후 스스로 질문해보세요: "이걸 읽은 사람이 바로 무엇을 해야 하는지 알 수 있는가?"
"예" → 좋은 UX라이팅. "모르겠다" → 더 구체적으로 다듬을 여지가 있습니다.
03  ·  Before / After

텍스트 한 줄이 바꾸는 경험

Before
"제출서류를 구비하여
기한 내 접수 요망"
❌ "기한"이 언제인지 없음
❌ "어디로" 접수해야 하는지 없음
❌ 읽고 나서 전화를 해야 함
After
"신청서를 작성해서
4월 15일(화) 오후 6시까지
sema@seoul.go.kr로 보내주세요."
✓ 읽으면 바로 행동 가능
✓ 날짜 · 방법 · 수신처 명확
✓ 문의 전화 감소
💡
좋은 UX라이팅은 읽고 나서 다음 행동이 보이는 글입니다.
화려한 문장보다, 한 줄이 명확한 글이 먼저입니다.
03  ·  버튼 텍스트

버튼 하나가 신뢰를 결정한다

❌ 피해야 할 버튼
확인 예 제출 삭제 OK
클릭 후 무슨 일이 일어나는지 모름
"확인"은 무엇을 확인하는 건지?
✓ 권장하는 버튼
예약 완료하기 참여 신청하기 예약 취소하기 PDF 다운로드
동사+목적어 = 행동이 명확
클릭 전에 결과를 예측할 수 있음
🔘
버튼 텍스트 공식: [동사] + [목적어] → "예약 취소하기" "신청서 제출하기"
최소 2단어. 클릭하면 어떤 일이 생기는지 클릭 전에 알 수 있어야 합니다.
04  ·  UX라이팅 3원칙

SeMA가 적용할 3가지 원칙

01
Clear
명확하게
읽은 후 "이게 무슨 말이지?"가 없도록. 한 문장에 하나의 메시지만. 전문용어·기술어 최소화.
02
Specific
구체적으로
날짜, 장소, 방법, 수신처를 명시. "곧" "빠른 시일" "기한 내" 대신 실제 날짜와 시간.
03
Human
인간적으로
시스템이 말하는 것처럼 들리지 않게. 오류 상황에서도 사람처럼. 딱딱한 경어보다 자연스러운 존댓말.
KRDS 공식 3대 원칙 연결 — 이 3원칙은 KRDS UX라이팅 가이드라인의 공식 원칙을 SeMA 현장 언어로 풀어쓴 것입니다: ① 무번역의 원칙(Clear)  ·  ② 정보핵심화의 원칙(Specific)  ·  ③ 심리적안전망의 원칙(Human)
